Wrong Java Version

When starting the game, you may see that it tries to start the game with Java 15/16, even though Java 8 is installed. To fix this, we need to edit some project settings.

1. Locating installed JREs

Right click on your project, and click on Properties. Next, go down to Java Build Path, and in the properties window, select the Libraries tab. Click JRE System Library and then click Edit on the left. Finally, click Installed JREs.

2. Changing build path

Now, click Add, select Standard VM and hit Next. Next, click Directory and locate the installation path of the Java 8 JRE. If you have downloaded OpenJDK as I have done in my video tutorials, you will find the path under C:\Program Files\AdoptOpenJDK\jdk-8.0.282.8-hotspot\jre. Once you have done that, click Finish. Then, tick the box thats next to the Java 8 JRE, and click Apply and Close. Click Alternate JRE, and select the Java 8 JDK from the dropdown. Finally, click Finish and then Apply and Close.

3. Refresh gradle

To rebuild gradle, right click on your project and go to Gradle > Refresh Gradle Project, and wait for the progress bar in the bottom right to finish.

Thats it! Good luck with modding :D

Mappings

Mappings are basically what defines a functions name. By default, in 1.16.5, forge uses Mojmaps, but my tutorials use MCP. MCP have less functions mapped, but they usually have more user friendly descriptions.

1. Changing mappings

If you would like to change, simple do the following steps: + Go to your mod's folder and open CMD. + Type gradlew -PUPDATE_MAPPINGS=20201028-1.16.3 -PUPDATE_MAPPINGS_CHANNEL=snapshot updateMappings and wait for the process to finish. + Locate your build.gradle file in your project. + Open the file and find <mappings channel: 'official', version: '1.16.5'>. + Replace it with <mappings channel: 'snapshot', version: '20201028-1.16.3'>. + Save the file. + Right click on your project and go to gradle>refresh. + Wait for the refresh to complete.

2. Using Mojmaps

Alternatively, you can use the bot in #bot-commands in our discord. Simply type !mcpmm <name_of_function> 1.16 in order to convert from the MCP used in my tutorials to the Mojmaps used by 1.16.5.

Exporting

In this guide I will be showing you how to export your mod so you can let other people play it. Let's begin!

Go to your mods folder and open CMD. Type gradlew build and wait for the process to finish. Your .jar file will be located in the build/libs folder. That's it! Make sure to test your mod before exporting and upload the jar onto CurseForge once you are done!

How to make a Food item

1. Basic Food

In order to make a food, we need to add a Food Builder to our Item Properties! So, in our ItemInit, where we initialize our item, lets add a basic item:

public static final RegistryObject<Item> EXAMPLE_FOOD ITEMS.register("example_food",
	() -> new Item(new Item.Properties().group(TutorialModItemGroup.TUTORIAL_MOD)));

Now, lets add a food builder to this item:

public static final RegistryObject<Item> EXAMPLE_FOOD ITEMS.register("example_food",
	() -> new Item(new Item.Properties().group(TutorialModItemGroup.TUTORIAL_MOD)
		.food(new Food.Builder()
		.hunger(4)
		.saturation(1.2f)
		.build()));

The food builder above gives the food a hunger value of 4 and a saturation value of 1.2. You can find the default values here.

2. Food with Effects

In order to add an effect to the food, we need to add a couple of things to the food builder:

public static final RegistryObject<Item> EXAMPLE_FOOD ITEMS.register("example_food",
	() -> new Item(new Item.Properties().group(TutorialModItemGroup.TUTORIAL_MOD)
		.food(new Food.Builder()
		.hunger(4)
		.saturation(1.2f)
		.effect(new EffectInstance(Effects.REGENERATION, 200, 0), 1f)
		.build()));

Lets break that down. effect takes two parameters, an EffectInstance and a float. The float value at the end, in this case 1f, is the probability that this effect will be given. The effect instance contains 3 values, the effect, in this case Effects.REGENERATION, the time in ticks it will last in this case 200 and the effect's strength (0 - 254). Note that in this case, 0 will give a strength of 1, 1 will give a strength of 2 ect...

How to make a Fuel Item

1. Item Class

First, lets create the class for an item! Lets start by creating a simple class (usually in common.items) that extends Item:

class ExampleFuelItem extends Item {
	public ExampleFuelItem (final Item.Properties properties) {
		super(properties);
	}
}

Then, lets override getBurnTime. This is a forge method, so its the same for any mappings!

class ExampleFuelItem extends Item {
	public ExampleFuelItem (final Item.Properties properties) {
		super(properties);
	}

	@Override
	public int getBurnTime(ItemStack itemStack) {
		return 14400; // The length of burning in ticks!
	}
}

Once we've returned the correct value, we can close and save the class!

2. Registering the Item

Next, lets go in our ItemInit and register the item like we would any other!

public static final RegistryObject<ExampleFuelItem> EXAMPLE_FUEL = ITEMS.register("example_fuel",
	() -> new ExampleFuelItem(new Item.Properties().group(TutorialModItemGroup.TUTORIAL_MOD)));

And that's it! We've created a custom fuel.

How To Use The Blit Method

So many people get confused when rendering the gui in a Screen class via blit() method. Here is the explanation of the parameters (I mapped all the params so you can easily understand):

IMPORTANT: Your png gui file haas to be 256x256 square and you dont need to use all of the pixels.

blit(MatrixStack matrixStack, int leftPos, int topPos, int x, int y, int xSize, int ySize)

  • matrixStack - just pass the MatrixStack value from renderBg().

  • leftPos and topPos : position (x, y) where the screen png texture file is being renderered on the game screen. Using the default value for leftPos and topPos in super class is recommended.

  • x , y : the position in your gui texture from which it will start to render (the top, left corner). Count from the very top left of the png file (it's 0, 0).

  • xSize and ySize: The size of your png texture. If you will start drawing at 5, 5 and your sizes were 50, 50 it will draw from 5, 5 to 55, 55.

REMEMBER : Param names doesn't affect to your code outside this method, they are just useable inside this method.
