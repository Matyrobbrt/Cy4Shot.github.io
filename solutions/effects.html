Effects

1. Creating An Effect

In order to create an effects we will need a new class for it. Usually, that class will go in common.effect. For this tutorial we will name the class TutorialEffect

We will make the class we just created extend net.minecraft.potion.Effect (or, in 1.17 net.minecraft.world.effect.MobEffect). We will create the default constructor, but remove the parameters of it. In the super we will need to pass in: 1 - The EffectType of our effect; 2 - The particle colour of our effect (if we want to use a hex value we will pass in 0xhexValue where hexValue is the hex value of the colour.):


public TutorialEffect() {
    super(EffectType.BENEFICIAL, 0x26ADFF);
}

Ok, now let's make our effect do something special! For the sake of this tutorial our effect will insta-kill the entity. We will start by overriding isDurationEffectTick and returning true:


@Override
public boolean isDurationEffectTick(int duration, int amplified) {
    return true;
}

And in order to execute what our effect does we can override applyEffectTick. Inside it we will execute the logic behind our effect. (Anything inside this method will be executed every tick):


@Override
public void applyEffectTick(@Nonnull LivingEntity entity, int amplifier) {
    entity.kill();
    // You can execute as much logic as you want here, but keep in mind: anything inside this method is executed every tick
}

2. Registering Effects

After we created our effect, we need to register it. In the core.init package we will create an EffectInit class. Inside this class we will put the following code:


public static final DeferredRegister<Effect> EFFECTS = DeferredRegister.create(ForgeRegistries.POTIONS, MainClass.MOD_ID);

public static final RegistryObject<TutorialEffect> TUTORIAL_EFFECT = EFFECTS.register("tutorial_effect", TutorialEffect::new);

Where MainClass.MOD_ID is a reference to the mod id from our main class, tutorial_effect is the registry name of our effect and TutorialEffect is our effect class.

Finally we will register the DeferredRegister inside EffectInit to the IEventBus from our main mod class by simply doing:


EffectInit.EFFECTS.register(bus);

3. Making an Effect Icon

In order for our effect to have an icon, we will need to have an 18x18 pixels .png texture inside assets.mod_id.textures.mob_effect (mod_id being your mod id) which has the same name as the effects' registry name. (In this case it will be named tutorial_effect)

Also, we should create a lang entry if we want our effect to have a display name. The value of the lang entry will be effect.mod_id.registry_name where mod_id is your mod id and registry_name is the effects' registry name. (In this case the registry name is tutorial_effect)
